# 事務系マニュアル

## 1. 利用者管理

個別支援計画の作成から本人への説明・署名取得まで、12ステップの詳細な流れを記載します。

---

### 1-1 個別支援計画の期限確認（詳細版）

**実施タイミング：** 毎週月曜日

**確認時間帯：** 業務開始時 8:30～9:00（朝礼前）

**手順：**

1. **Googleカレンダーを開く**
   - 個別支援計画の期限管理用カレンダーを確認

2. **期限がある利用者を確認**
   - 形式：「★個別支援計画期限　7/1藤井さん」

3. **期限の緊急度を判定**
   - **1週間以内（緊急）** → 優先順位を上げる
   - **2～4週間以内（注意）** → 通常スケジュール
   - **1ヶ月以上先** → 予定として認識

4. **緊急案件の対応**
   - 期限が1週間以内の場合、その週のスケジュールに「個別支援計画対応（〇〇さん）」と記入
   - 朝礼で関連スタッフに伝達

5. **朝礼での報告**
   - 月曜日の朝礼で、その週の期限切れが近い利用者を報告

---

### 1-2 モニタリング面談日の調整（詳細版）

**実施タイミング：** 個別支援計画の期限から20日前から

**計算方法：**
```
個別支援計画期限：7/1
↓ 3ヶ月遡る
公開日：4/1
↓ 20日前から調整開始
調整開始日：6/11
```

**手順：**

1. 面談予定日を逆算する
2. Googleカレンダーに調整予定を記入：「モニタリング面談調整対象（〇〇さん）」
3. 利用者に面談日の確認を取る（こちらの都合を伝える）
4. 提案する面談日：期限の10～15日前
5. 面談日を決定してカレンダーに登録

---

### 1-3 利用者様の情報収集（詳細版）

**実施タイミング：** モニタリング面談の基本 2週間前から

**情報収集方法：**

- **方法 A：Notta（音声メモ）を使用** - 朝礼終了後、「〇〇さんの2週間の変化や課題について、ご意見ありますか？」と質問して音声記録
- **方法 B：ノートにメモ** - 日付・支援担当者の名前も記入

**情報源：** すべてのスタッフがランダムに答える（複数視点から把握するため、3～5名程度）

**情報の管理：**
- Notta：毎日のファイルを「〇〇さん_6月」フォルダに保存、テキスト版もダウンロード
- ノート：日付ごとに項目分けして記入

**情報の整理：**
- 面談 3～4 日前に収集した情報をまとめる
- 共通のテーマ（課題・成長点）を抽出
- Google Docs に「〇〇さん_モニタリング情報」として整理

---

### 1-4 モニタリング面談（詳細版）

**場所：** 面談室

**所要時間：** 30分

**目的：** 前回の個別支援計画を見ながら3ヶ月の振り返りと今後3ヶ月の確認（聞き取りがメイン）

**事前準備：**
- 前回の個別支援計画書
- 過去3ヶ月間の支援記録
- スタッフの意見（Google Docs）
- 筆記用具・メモ帳

**面談の流れ（30分）：**

| 時間 | 内容 | 備考 |
| --- | --- | --- |
| 0～2分 | 挨拶・面談の目的説明 | 「本日は3ヶ月の振り返りをしたいと思います」 |
| 2～15分 | 前回計画の振り返り | 「前回立てた目標、達成できましたか？」 |
| 15～25分 | 今後3ヶ月の取り組み確認 | 「今後、何に取り組みたいですか？」 |
| 25～30分 | まとめと次のステップ説明 | 「では、〇〇に取り組んでいきましょう」 |

**聞き取りのポイント：**
- オープンクエスチョン（「どうでしたか？」）を主に使う
- 利用者の言葉をそのまま記録
- 面談直後、メモを整理して Google Docs に転記

---

### 1-5 モニタリング記録の作成（詳細版）

**実施タイミング：** モニタリング面談後、1週間以内

**使用ツール：** 凸凹システム（専用テンプレート）

**入力項目：**
- 利用者名：自動入力
- モニタリング実施日：面談実施日
- 面談場所：面談室
- 前回計画の達成状況：【要点のみ】面談時のメモから抜き出す
- 現在の状況・課題：【要点のみ】利用者の現状
- 次期計画への方針：【要点のみ】今後の取り組み
- 作成日、作成者

**入力例：**
```
【前回計画の達成状況】
○ 朝の準備を自分でできるようになった
✕ プレゼンテーションスキルの習得は途中

【現在の状況・課題】
- 対人関係が改善し、スタッフとの会話が増えた
- 新しいタスクに取り組む際、不安感がある

【次期計画への方針】
- プレゼンテーションスキルの継続学習
- 新タスク習得時のサポート方法の工夫
```

---

### 1-6 個別支援計画原案の作成（詳細版）

**進捗目安：** 基本は週2名分、最大3名まで対応可能

**使用ツール：** Google Docs + Gemini の専用 Gem

**手順：**

1. **Gemini 専用 Gem で初期ドラフト生成**
   - 「個別支援計画作成 Gem」を開く
   - 対象利用者の情報を入力（利用者名、前回計画、モニタリング結果、利用者の希望、現在の課題）

2. **Google Docs に出力内容をコピー**
   - ファイル名：「個別支援計画原案_〇〇さん_令和8年」
   - 共有フォルダ「/個別支援計画/2026年度/」に保存

3. **テンプレートに合わせて編集**
   - ご本人・ご家族の考え
   - 生活全般の課題（3つの課題）
   - 総合的な支援方針
   - 備考
   - 長期目標
   - 短期目標1～3（各々に「具体的な手立て・達成時期」を含む）

4. **編集時のポイント**
   - 利用者の言葉を活かす
   - SMART 目標に統一（Specific/Measurable/Achievable/Relevant/Time-bound）

5. **前回計画と比較して確認**
   - 達成できた項目は長期目標に反映
   - 達成できない項目は短期目標に継続
   - 新しい課題は新規追加

---

### 1-7 アセスメントシートの更新（詳細版）

**実施タイミング：** 3ヶ月ごと（個別支援計画作成時）

**重要：** 日付は個別支援計画原案作成より前の日にちで作成

**使用ツール：** Google Sheets（テンプレート）+ 印刷してファイリング

**更新項目：**

| 項目 | 内容 | 更新ポイント |
| --- | --- | --- |
| 基本情報 | フリガナ、氏名、生年月日、年齢 | 年齢のみ更新 |
| 医療情報 | 診断名、診断日、障害者手帳、障害年金、知能検査 | 新たな検査があれば追記 |
| 生活情報 | 生活習慣、自活状況、家族構成 | 3ヶ月での変化を反映 |
| 職歴・就職 | 職歴、就職希望時期、雇用形態、希望内容 | モニタリング面談で確認した内容に更新 |
| 特性・課題 | 特性、就職に向けた課題 | **最も重要。スタッフ聞き取り + 本人発言で詳しく更新** |
| 主治医の意見 | 医師の診察時のコメント | 定期的な医師の診察時に確認 |

**手順：**

1. 前回のシートをコピーして新規作成
2. 各項目を確認・更新
3. 日付を記入（個別支援計画原案作成日より前）
4. Google Sheets を PDF に変換
5. PDF を印刷してファイリング（モニタリング記録の上）

---

### 1-8 策定会議の実施（詳細版）

**参加者：** 全スタッフ（基本固定メンバー）

**開催日時：** 毎週水曜日 16:00～18:00（2時間）

**調整日程：** 必要に応じて月曜日・金曜日 16:00～18:00

**目的：** モニタリング記録と個別支援計画原案をスタッフ全体で協議し、支援内容を精査する

**事前準備（水曜日の3～4日前）：**

1. 対象利用者を決定（3～5名程度）
2. LINE WORKS で「支援会議」チャネルに事前通知
3. Google カレンダーで招待状を送信（全スタッフへ）
4. 会議資料を共有フォルダに保存

**会議当日の流れ（16:00～18:00）：**

| 時間 | 内容 | 詳細 |
| --- | --- | --- |
| 16:00～16:05 | 会議開始・本日の対象利用者確認 | 本日の予定を説明 |
| 16:05～16:35 | 【利用者 1】協議 | (30分) |
| 16:35～17:05 | 【利用者 2】協議 | (30分) |
| 17:05～17:35 | 【利用者 3】協議 | (30分) |
| 17:35～18:00 | まとめ・今後の対応 | 議事録確認・次回の予定 |

**各利用者の協議進行（1名あたり 30分）：**

1. 原案の概要説明（5分）
2. モニタリング結果の共有（5分）
3. スタッフからの意見聴取（15分）
4. 修正内容のまとめ（5分）

**会議終了後：**
- 議事録を整理
- 修正内容をリストアップ
- LINE WORKS で結果を報告

---

### 1-9 策定会議議事録の作成（詳細版）

**実施タイミング：** 策定会議実施後

**使用ツール：** Google Docs（1-8 でリアルタイム記録したものをそのまま使用）

**保管方法：** 印刷して利用者の個別ファイルにファイリング

**ファイリング位置：** 個別支援計画原案と個別支援計画の間

**手順：**

1. Google Docs の議事録を確認・整理
2. Google Docs を PDF に変換（「ファイル」→「ダウンロード」→「PDF」）
3. PDF を印刷（A4サイズ、カラー推奨、片面印刷）
4. 個別ファイルに挿入
5. 付箋で位置を明記：「支援会議議事録（令和8年6月19日）」
6. Google Docs にファイリング完了を記載
7. LINE WORKS で完了報告

**個別ファイルの正しい順序：**

```
1. モニタリング記録
2. アセスメント
3. 計画原案
4. 【策定会議議事録】← ここに挿入
5. 本人署名済計画（公開版）
```

---

修正・追記後に本番に変更し、2枚印刷する。

4. 議事録

## 2. 月次報告書

### 2-1 月次報告書の作成・提出（詳細版）

**実施タイミング：** 毎月15日までに

**対象：** 事業所を利用されている全員

**作成方法：** Google の支援経過報告書 Gem を使用

**参考データ：** LINE WORKS（スタッフ共有グループLINE） / 社内共有LINE（コピペ＋面談ログ） / KPT（1か月の振り返り） / 予実管理シート（毎日の予定・やったこと・気づき・スタッフフィードバック）

**提出先：** 関係機関への訪問時に提出（訪問頻度：月1回程度）

**訪問対象：** 1人のスタッフが複数利用者を担当

---

**手順：**

#### ステップ 1: 参考データの収集（月初～中旬）

1. **LINE WORKS から情報収集**
   - 「スタッフ共有グループLINE」チャネルを確認
   - その月の全体共有事項をメモ

2. **社内共有 LINE から情報抽出**
   - スタッフが共有した利用者に関する情報をコピペ
   - 面談ログから利用者の発言・気づきを抽出

3. **Google Sheets の予実管理シート を確認**
   - 毎日の予定・やったこと・気づき・スタッフフィードバックを収集

4. **KPT 振り返りシートを確認**
   - Keep（続ける）/ Problem（改善）/ Try（試す）を確認

#### ステップ 2: Google Docs で支援経過報告書 Gem にデータを入力

1. **支援経過報告書 Gem にアクセス**
   - 対象利用者のデータを入力

2. **Gem に入力するパラメータ：**
   - 報告書作成日 / 利用者名 / 現在の就労準備度
   - 近況報告の主要トピック / 具体的な支援内容と効果
   - 事業所が把握する現在の課題 / 今後の計画

#### ステップ 3: 生成された報告書を確認・修正

1. Google Docs で出力内容を確認
2. 具体的な数字・訓練内容が記載されているか確認
3. 修正が必要な場合は Google Docs でテキストを編集

#### ステップ 4: PDF に変換・印刷

1. Google Docs を PDF に変換
2. A4 サイズで印刷（カラー推奨）

#### ステップ 5: ファイリング

1. 共有フォルダに保存：`/月次報告書/2026年6月/支援経過報告書_〇〇さん.pdf`
2. 利用者の個別ファイルに保管

#### ステップ 6: 関係機関への訪問・提出

1. 1人のスタッフが複数利用者を担当
2. 月1回程度のペースで関係機関を訪問
3. 該当利用者の月次報告書を提出
4. 関係機関からの意見・助言を聴取
5. 帰所後、スタッフと情報を共有

---

## 3. 緊急・災害対応

自然災害や緊急事態が発生した際の利用者連絡と対応フロー

### 3-1 台風・大雨時の訓練実施判断と通知（詳細版）

**判断タイミング：** 前夜～当日朝（決定時刻を事前に設定）

**判断基準：** 気象庁の警報レベル・交通状況を総合的に判断

**対応パターン：**

#### パターンA：判断中（実施可否未定の通知）

以下のメールを該当利用者へ送信：

```
件名: 【重要】[台風/気象]の影響による訓練実施について

利用者の皆さん

いつもお世話になっております
ディーキャリア○○オフィスです。

いつも当事業所にご連絡いただき、心より感謝申し上げます。皆様のご理解とご協力のおかげで、私たち日々充実した支援を提供させていただいております。

さて、明日（日付）に予定しております訓練についてですが、現在接近中の[台風/気象現象]の影響を見極め、安全面を最優先に考え、実施を検討しております。天候や交通状況を慎重に確認した上で、訓練の実施可否を決定いたします。

最終的な判断は明日（日付）（決定時刻）までお知らせいたしますので、権利ですが、メールやお電話でのご案内をご確認いただけますようお願いいたします。

また、明日（日付）の〇時〇分以降はお電話でもご連絡が可能ですので、ご質問やご不明な点がございましたら、どうぞお気軽にご連絡ください。

■連絡先
ディーキャリア○○オフィス：（電話番号）
　※連絡可能時間：○○：○○以降

明日の訓練が中止となった場合でも、今後のスケジュールに影響が出ないよう対応いたします。

どうぞよろしくお願いいたします。

（事業所名）：
（電話番号）：
（メールアドレス）：
```

#### パターンB：実施決定（時間変更あり）

以下のメールを該当利用者へ送信：

```
件名: 【お知らせ】明日（日付）の訓練は午前中のみ実施

利用者の皆さん

いつもお世話になっております。
ディーキャリア○○オフィスです。

いつも当事業所にご連絡いただき、心より感謝申し上げます。皆様のご理解とご協力のおかげで、私たち日々充実した支援を提供させていただいております。

さて、明日（日付）に予定している訓練について、[台風/気象現象]の影響を考慮した結果、安全面を第一に考え、『午前中のみ実施する』対応させていただきます。

訓練は（開始時間）から（終了時間）まで行われますので、通常通りご参加いただけます。ご安心ください。

また、明日（日付）の〇時〇分以降はお電話でもご連絡が可能ですので、ご質問やご不明な点がございましたら、どうぞお気軽にご連絡ください。

■連絡先
ディーキャリア○○オフィス：（電話番号）
　※連絡可能時間：○○：○○以降

今後の安全と健康を最優先に考えた判断となりますので、どうぞご理解いただけますようお願い申し上げます。

どうぞよろしくお願いいたします。

（事業所名）：
（電話番号）：
（メールアドレス）：
```

**実施手順：**

1. 前夜までに実施判断の時刻を設定（例：翌朝 6:00）
2. 設定時刻に、気象庁警報・交通情報を確認
3. パターンA または B のメールをコピー、日付・時刻・内容を修正
4. LINE WORKS で「利用者連絡」チャネルに投稿
5. メールで全利用者に配信
6. 決定内容を Google カレンダーに記載（キャンセル or 時間変更で表記）

**保管方法：**
- メール文は Google Drive「/緊急対応」フォルダに「日付_気象対応メール」として保存
- 履歴として全通知を記録

## 4. 新規利用者対応

利用者が新たに事業所での利用を開始する際の契約手続き、初期評価、物理的設備対応まで

### 4-1 利用者契約手続き（詳細版）

**対応者：** 事業責任者（主）+ 事務スタッフ

**流れ：**

#### ステップ 1: 紹介・問い合わせ受付

**紹介経路：**
- 関係機関からの紹介（相談支援専門員、ハローワーク、医療機関など）
- 直接本人からの問い合わせ（メール、電話）

**初期対応：**
- メールまたは電話で基本情報を確認
- 来所日時を調整
- 初回来所時の持参書類を案内（身分証、障害者手帳等）

#### ステップ 2: 来所時インテーク（初回面談前）

**実施内容：**
1. インテークシートに基本情報を記入（本人記入）
   - 氏名、生年月日、住所、連絡先
   - 緊急連絡先
   - 障害手帳・年金情報
   - 現在の医療機関

2. 事業所のリーフレットを配布
3. 施設案内（簡潔に）

#### ステップ 3: 初回面談（事業責任者が主に担当）

**場所：** 面談室

**所要時間：** 60～90分

**事前準備：**
- インテークシート
- 支援サービスの説明資料
- 利用契約書・関連書類

**面談内容：**

| 項目 | 詳細 |
| --- | --- |
| 事業所の説明 | サービス内容、訓練プログラム、スケジュール |
| 本人の希望確認 | 就労・訓練の目標、希望する環境・内容 |
| 通勤方法の確認 | 公共交通、送迎の要否 |
| 利用に必要な行政手続き | 申請が必要な場合、スタッフが補助することを説明 |
| 服薬・医療情報 | 現在の医学的サポートが必要かどうか |
| アレルギー・食事対応 | 食物アレルギーの有無、食事提供の確認 |
| 契約内容説明 | 利用期間、利用料金、休止・退所時の手続き |

**面談後の説明：**
- 利用開始前に体験期間を設けることを案内（複数日）

#### ステップ 4: 行政手続きの補助・確認

**対応内容：**

| 手続き | 実施主体 | タイミング |

| その他福祉制度相談 | スタッフが情報提供 | 必要に応じて |

**スタッフの対応：**
1. 本人が手続きを自分で行える場合 → 情報提供のみ
2. 支援が必要な場合 → 一緒に手続きを実施
3. 必要に応じて相談支援専門員に相談

#### ステップ 5: 契約書類の説明と署名

**署名手順：**
1. 各書類を1つずつ説明
2. 本人が理解した上で署名

4. 原本と控えを分ける

### 4-3 パソコン貸出手続き（詳細版）

**実施タイミング：** パソコン貸与直前

**対応者：** 支援員

**事前準備：**
- Google Drive の共有フォルダ「/様式・テンプレート」から「パソコン貸出誓約書」テンプレートを取得

**手順：**

1. **誓約書の印刷**
   - テンプレートをダウンロード
   - A4用紙に片面印刷（カラー推奨）

2. **利用者への説明**
   - 以下の重要事項を説明：
     - 業務目的のみの使用
     - セキュリティ対策（パスワード設定等）
     - 持ち出し時の許可申請
     - 返却義務
     - 損傷時の弁償責任

3. **署名・記入**
   - 利用者に以下を記入させる：
     - 日付（年月日）
     - 住所
     - 電話番号
     - 署名＆印（印鑑がない場合は署名のみ）

4. **担当者記入欄を完成**
   - 支援員が以下を記入：
     - パソコンの型番
     - シリアル番号
     - 付属品（マウス・ACアダプタ等）
     - 貸出期間（開始日～終了日）
     - 担当者氏名

5. **保管**
   - 署名済み誓約書を印刷版のまま利用者の個別ファイルにファイリング
   - ファイリング位置：利用者基本情報の直後

6. **利用者への配布**
   - コピーを1部利用者に渡す（利用者控え）

**誓約書の内容（パソコン使用ルール）：**

| ルール | 詳細 |
| --- | --- |
| 使用目的 | 業務関連作業のみ。私的利用は禁止 |
| 使用場所 | 承認された場所のみ。持ち出しは事前許可が必要 |
| セキュリティ | パスワード設定、定期的なウイルス対策ソフトの実行 |
| ソフトウェア | インストールは事前許可が必要。違法ソフトは使用禁止 |
| データ管理 | 業務に無関係なデータは保存禁止。定期的にバックアップ実施 |
| 返却義務 | 貸出期間終了時または事業所からの要請時に速やかに返却 |
| 損害賠償 | 故意または過失による損傷・紛失の場合は修理・再調達費用を弁償 |

**パソコン返却時の対応：**
- 誓約書に返却日を追記
- 返却時の状態をチェック（損傷がないか確認）
- 個別ファイルに「返却完了」の日付を記入
