# 事務系マニュアル

## 1. 利用者管理

個別支援計画の作成から本人への説明・署名取得まで、12ステップの詳細な流れを記載します。

---

### 1-1 個別支援計画の期限確認（詳細版）

**実施タイミング：** 毎週月曜日

**確認時間帯：** 業務開始時 8:30～9:00（朝礼前）

**手順：**

1. **Googleカレンダーを開く**
   - 個別支援計画の期限管理用カレンダーを確認

2. **期限がある利用者を確認**
   - 形式：「★個別支援計画期限　7/1藤井さん」

3. **期限の緊急度を判定**
   - **1週間以内（緊急）** → 優先順位を上げる
   - **2～4週間以内（注意）** → 通常スケジュール
   - **1ヶ月以上先** → 予定として認識

4. **緊急案件の対応**
   - 期限が1週間以内の場合、その週のスケジュールに「個別支援計画対応（〇〇さん）」と記入
   - 朝礼で関連スタッフに伝達

5. **朝礼での報告**
   - 月曜日の朝礼で、その週の期限切れが近い利用者を報告

---

### 1-2 モニタリング面談日の調整（詳細版）

**実施タイミング：** 個別支援計画の期限から20日前から

**計算方法：**
```
個別支援計画期限：7/1
↓ 3ヶ月遡る
公開日：4/1
↓ 20日前から調整開始
調整開始日：6/11
```

**手順：**

1. 面談予定日を逆算する
2. Googleカレンダーに調整予定を記入：「モニタリング面談調整対象（〇〇さん）」
3. 利用者に面談日の確認を取る（こちらの都合を伝える）
4. 提案する面談日：期限の10～15日前
5. 面談日を決定してカレンダーに登録

---

### 1-3 利用者様の情報収集（詳細版）

**実施タイミング：** モニタリング面談の基本 2週間前から

**情報収集方法：**

- **方法 A：Notta（音声メモ）を使用** - 朝礼終了後、「〇〇さんの2週間の変化や課題について、ご意見ありますか？」と質問して音声記録
- **方法 B：ノートにメモ** - 日付・支援担当者の名前も記入

**情報源：** すべてのスタッフがランダムに答える（複数視点から把握するため、3～5名程度）

**情報の管理：**
- Notta：毎日のファイルを「〇〇さん_6月」フォルダに保存、テキスト版もダウンロード
- ノート：日付ごとに項目分けして記入

**情報の整理：**
- 面談 3～4 日前に収集した情報をまとめる
- 共通のテーマ（課題・成長点）を抽出
- Google Docs に「〇〇さん_モニタリング情報」として整理

---

### 1-4 モニタリング面談（詳細版）

**場所：** 面談室

**所要時間：** 30分

**目的：** 前回の個別支援計画を見ながら3ヶ月の振り返りと今後3ヶ月の確認（聞き取りがメイン）

**事前準備：**
- 前回の個別支援計画書
- 過去3ヶ月間の支援記録
- スタッフの意見（Google Docs）
- 筆記用具・メモ帳

**面談の流れ（30分）：**

| 時間 | 内容 | 備考 |
| --- | --- | --- |
| 0～2分 | 挨拶・面談の目的説明 | 「本日は3ヶ月の振り返りをしたいと思います」 |
| 2～15分 | 前回計画の振り返り | 「前回立てた目標、達成できましたか？」 |
| 15～25分 | 今後3ヶ月の取り組み確認 | 「今後、何に取り組みたいですか？」 |
| 25～30分 | まとめと次のステップ説明 | 「では、〇〇に取り組んでいきましょう」 |

**聞き取りのポイント：**
- オープンクエスチョン（「どうでしたか？」）を主に使う
- 利用者の言葉をそのまま記録
- 面談直後、メモを整理して Google Docs に転記

---

### 1-5 モニタリング記録の作成（詳細版）

**実施タイミング：** モニタリング面談後、1週間以内

**使用ツール：** 凸凹システム（専用テンプレート）

**入力項目：**
- 利用者名：自動入力
- モニタリング実施日：面談実施日
- 面談場所：面談室
- 前回計画の達成状況：【要点のみ】面談時のメモから抜き出す
- 現在の状況・課題：【要点のみ】利用者の現状
- 次期計画への方針：【要点のみ】今後の取り組み
- 作成日、作成者

**入力例：**
```
【前回計画の達成状況】
○ 朝の準備を自分でできるようになった
✕ プレゼンテーションスキルの習得は途中

【現在の状況・課題】
- 対人関係が改善し、スタッフとの会話が増えた
- 新しいタスクに取り組む際、不安感がある

【次期計画への方針】
- プレゼンテーションスキルの継続学習
- 新タスク習得時のサポート方法の工夫
```

### 1-6 個別支援計画原案の作成（詳細版）

**進捗目安：** 基本は週2名分、最大3名まで対応可能

**使用ツール：** Google Docs + Gemini の専用 Gem

**手順：**

1. **Gemini 専用 Gem で初期ドラフト生成**
   - 「個別支援計画作成 Gem」を開く
   - 対象利用者の情報を入力（利用者名、前回計画、モニタリング結果、利用者の希望、現在の課題）

2. **Google Docs に出力内容をコピー**
   - ファイル名：「個別支援計画原案_〇〇さん_令和8年」
   - 共有フォルダ「/個別支援計画/2026年度/」に保存

3. **テンプレートに合わせて編集**
   - ご本人・ご家族の考え
   - 生活全般の課題（3つの課題）
   - 総合的な支援方針
   - 備考
   - 長期目標
   - 短期目標1～3（各々に「具体的な手立て・達成時期」を含む）

4. **編集時のポイント**
   - 利用者の言葉を活かす
   - SMART 目標に統一（Specific/Measurable/Achievable/Relevant/Time-bound）

5. **前回計画と比較して確認**
   - 達成できた項目は長期目標に反映
   - 達成できない項目は短期目標に継続
   - 新しい課題は新規追加

---

### 1-7 アセスメントシートの更新（詳細版）

**実施タイミング：** 3ヶ月ごと（個別支援計画作成時）

**重要：** 日付は個別支援計画原案作成より前の日にちで作成

**使用ツール：** Google Sheets（テンプレート）+ 印刷してファイリング

**更新項目：**

| 項目 | 内容 | 更新ポイント |
| --- | --- | --- |
| 基本情報 | フリガナ、氏名、生年月日、年齢 | 年齢のみ更新 |
| 医療情報 | 診断名、診断日、障害者手帳、障害年金、知能検査 | 新たな検査があれば追記 |
| 生活情報 | 生活習慣、自活状況、家族構成 | 3ヶ月での変化を反映 |
| 職歴・就職 | 職歴、就職希望時期、雇用形態、希望内容 | モニタリング面談で確認した内容に更新 |
| 特性・課題 | 特性、就職に向けた課題 | **最も重要。スタッフ聞き取り + 本人発言で詳しく更新** |
| 主治医の意見 | 医師の診察時のコメント | 定期的な医師の診察時に確認 |

**手順：**

1. 前回のシートをコピーして新規作成
2. 各項目を確認・更新
3. 日付を記入（個別支援計画原案作成日より前）
4. Google Sheets を PDF に変換
5. PDF を印刷してファイリング（モニタリング記録の上）

---

### 1-8 策定会議の実施（詳細版）

**参加者：** 全スタッフ（基本固定メンバー）

**開催日時：** 毎週水曜日 16:00～18:00（2時間）

**調整日程：** 必要に応じて月曜日・金曜日 16:00～18:00

**目的：** モニタリング記録と個別支援計画原案をスタッフ全体で協議し、支援内容を精査する

**事前準備（水曜日の3～4日前）：**

1. 対象利用者を決定（3～5名程度）
2. LINE WORKS で「支援会議」チャネルに事前通知
3. Google カレンダーで招待状を送信（全スタッフへ）
4. 会議資料を共有フォルダに保存

**会議当日の流れ（16:00～18:00）：**

| 時間 | 内容 | 詳細 |
| --- | --- | --- |
| 16:00～16:05 | 会議開始・本日の対象利用者確認 | 本日の予定を説明 |
| 16:05～16:35 | 【利用者 1】協議 | (30分) |
| 16:35～17:05 | 【利用者 2】協議 | (30分) |
| 17:05～17:35 | 【利用者 3】協議 | (30分) |
| 17:35～18:00 | まとめ・今後の対応 | 議事録確認・次回の予定 |

**各利用者の協議進行（1名あたり 30分）：**

1. 原案の概要説明（5分）
2. モニタリング結果の共有（5分）
3. スタッフからの意見聴取（15分）
4. 修正内容のまとめ（5分）

**会議終了後：**
- 議事録を整理
- 修正内容をリストアップ
- LINE WORKS で結果を報告

---

### 1-9 策定会議議事録の作成（詳細版）

**実施タイミング：** 策定会議実施後

**使用ツール：** Google Docs（1-8 でリアルタイム記録したものをそのまま使用）

**保管方法：** 印刷して利用者の個別ファイルにファイリング

**ファイリング位置：** 個別支援計画原案と個別支援計画の間

**手順：**

1. Google Docs の議事録を確認・整理
2. Google Docs を PDF に変換（「ファイル」→「ダウンロード」→「PDF」）
3. PDF を印刷（A4サイズ、カラー推奨、片面印刷）
4. 個別ファイルに挿入
5. 付箋で位置を明記：「支援会議議事録（令和8年6月19日）」
6. Google Docs にファイリング完了を記載
7. LINE WORKS で完了報告

**個別ファイルの正しい順序：**

```
1. モニタリング記録
2. アセスメント
3. 計画原案
4. 【策定会議議事録】← ここに挿入
5. 本人署名済計画（公開版）
```

---

### 1-10 個別支援計画原案を本番に変更

修正・追記後に本番に変更し、2枚印刷する。

計画を説明し、修正確認、署名取得、本人控え提供を実施する。

4. 議事録
5. 本人署名済計画
